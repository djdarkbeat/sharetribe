Overview

Middleware for creating routes.js bundle.

Usage:

config.middleware.use JsRoutes::Middleware

By default the middleware is watching the `config/routes.rb` file. If you have splitted your routes file into multiple files, you need to pass the files to the middleware in `route_files` option:

config.middleware.use JsRoutes::Middleware, route_files: [

Rails.root.join('config/routes.rb'),
Rails.root.join('config/routes_admin.rb')

]

### Caching

The middleware implements a simple cache: It saves the times when the files were changed to the 'tmp/cache/routes.yml' file.

To clean the cache, run:

`rake tmp:cache:clear`

The cache is cleared everytime you restart the Rails server.

### Inspiration

The implementation is heavily inspired by i18n-js middleware: github.com/fnando/i18n-js/
